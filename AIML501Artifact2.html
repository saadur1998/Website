Components — Definitions & Explanations

Key neural network components and their roles.

Layers

Definition: Stacked groups of neurons arranged as input, hidden, and output layers.

Role: Organize computation stages; each layer transforms its inputs for the next layer.

Neurons

Definition: Computational units that take inputs, apply weights and bias, and produce an output via an activation function.

Role: Perform localized computation and introduce nonlinearity when paired with activation functions.

Weights

Definition: Trainable parameters that scale inputs between neurons.

Role: Determine the influence of one neuron on another; updated during training to reduce loss.

Activation Functions

Definition: Functions applied to a neuron's weighted sum (e.g., sigmoid, ReLU, tanh, softmax).

Role: Introduce nonlinearity allowing networks to model complex relationships; softmax converts outputs to probabilities for classification.

Loss Functions

Definition: A scalar measure of error between predicted outputs and true targets (e.g., MSE, cross-entropy).

Role: Guides training by providing a quantity to minimize via optimization algorithms.

Optimization Algorithms

Definition: Methods that update network weights to minimize the loss (e.g., SGD, Adam, RMSProp).

Role: Adjust weights iteratively using gradients to converge to a (local) minimum of the loss function.

Assignment Reflection

This artifact documents core neural network components and a visual presentation designed for instructors.

Creating this neural network visual presentation deepened my practical understanding of how model components interact. Building the diagram placeholder and composing concise component descriptions forced me to distill complex concepts layers, neurons, weights, activation functions, loss functions, and optimizers, into clear, teachable statements. That exercise highlighted the balance between precision and readability required for effective instructional artifacts.

A key insight was how visualization makes abstract operations concrete: showing data flow across layers clarifies where weights are applied, where activations introduce nonlinearity, and where the loss is computed for optimization. This helped me better appreciate architectural choices (e.g., depth vs width, activation selection) and practical trade-offs like interpretability, training stability, and computational cost.

Overall, the process reinforced core concepts and improved my ability to communicate them visually, a valuable skill for both teaching and presenting model architecture choices in real projects.
