Introduction

Artifact configuration: AIML501 Artifact 1 was created October, 2025. This page is a portfolio artifact for the course and is only visible to the individual with the link. It cannot be access from the main website.

How to read the infographic: Cards are grouped under "Supervised" and "Unsupervised & Generative". Each card includes Algorithm Type, Application Domains (Tabular Data, Computer Vision, NLP, Generative AI), Example Applications, and a short explanation so you can compare assumptions and suitable use cases.

Supervised Algorithms

Algorithms that require labeled data for training (classification / regression).

Linear Regression

Type: Supervised (Regression)

Domains: Tabular Data

Examples: Predicting housing prices, sales forecasting

How it works: Fits a linear function to minimize the difference between predicted and actual continuous values (least squares).

Logistic Regression

Type: Supervised (Classification)

Domains: Tabular Data, NLP

Examples: Email spam detection, binary classification

How it works: Models the probability of a class using the logistic (sigmoid) function on a linear combination of inputs.

Decision Trees

Type: Supervised (Classification / Regression)

Domains: Tabular Data, Some CV

Examples: Credit scoring, simple image-based heuristics

How it works: Recursively splits data on feature thresholds to create a tree of decisions that predict target values.

Random Forest

Type: Supervised (Ensemble)

Domains: Tabular Data, CV

Examples: Feature-rich classification, medical diagnosis

How it works: Builds many decision trees on bootstrapped samples and averages/votes predictions to reduce overfitting and variance.

Support Vector Machine (SVM)

Type: Supervised (Classification / Regression)

Domains: Tabular Data, CV

Examples: Handwritten digit recognition, margin-based classification

How it works: Finds a decision boundary (hyperplane) that maximizes the margin between classes; can use kernels for non-linear separation.

Unsupervised & Generative

Algorithms that do not require labeled targets or are used for generative modeling.

K-Means Clustering

Type: Unsupervised (Clustering)

Domains: Tabular Data, CV

Examples: Customer segmentation, image color quantization

How it works: Partitions data into K clusters by assigning points to nearest centroid and updating centroids iteratively.

Principal Component Analysis (PCA)

Type: Unsupervised (Dimensionality Reduction)

Domains: Tabular Data, CV

Examples: Feature compression, visualization (2D/3D)

How it works: Finds orthogonal directions (principal components) that explain the most variance and projects data onto fewer dimensions.

K-Nearest Neighbors (KNN)

Type: Supervised (Instance-based)

Domains: Tabular Data, CV

Examples: Simple classification, recommendation by similarity

How it works: Predicts a label by majority vote (or average) of the K nearest training examples in feature space.

Neural Networks (Deep Learning)

Type: Supervised / Unsupervised / Generative

Domains: CV, NLP, Tabular Data, Generative AI

Examples: Image classification (CNNs), language models (RNNs/Transformers), GANs for image synthesis

How it works: Composed of layers of parameterized neurons that transform inputs via learned weights and nonlinearities; trained with gradient-based optimization.

Legend: Cards are grouped visually by algorithm type. Use these cards to compare domains and how algorithms differ in assumptions and outputs.

Assignment Reflection

This artifact presents a visual framework covering 8â€“10 machine learning algorithms.

In creating this visual framework I focused on clarity and comparability across algorithms. I selected ten algorithms that represent a broad cross-section of machine learning techniques: classical regression and classification methods, tree-based and ensemble methods, instance-based learners, clustering and dimensionality reduction, and deep learning models. For each algorithm I included the algorithm type (supervised or unsupervised), typical application domains (Tabular Data, Computer Vision, NLP, Generative AI), concise real-world examples, and a short explanation of how the method operates.

The main challenge was balancing brevity with meaningful description: each card had to be short enough to read at a glance while still conveying the algorithm's assumptions and typical uses. To address this I grouped algorithms visually (Supervised vs Unsupervised & Generative) so readers can immediately compare which techniques require labels and which do not.

From a learning perspective I reinforced why certain algorithms are preferred for specific domains. For example, tree-based ensembles like Random Forests are robust on heterogeneous tabular data, convolutional neural networks excel at spatial pattern recognition in images, and transformer-based architectures are dominant in NLP and large-scale generative tasks. I also reflected on practical trade-offs: model interpretability (linear/logistic models, decision trees) versus predictive power (deep neural networks), training cost, and sensitivity to data scale and quality.
